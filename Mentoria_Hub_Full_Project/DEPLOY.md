# Публикация Mentoria Hub в интернете

Проект подготовлен для деплоя из GitHub на Render как один Node.js-сервис с постоянным диском. После деплоя Render выдаст публичный адрес вида `https://mentoria-hub-....onrender.com`.

## Почему не GitHub Pages

GitHub Pages размещает только статические файлы. Регистрация, защищённые сессии, общие чаты, прогресс и админ-панель требуют работающего сервера и постоянного хранилища.

## Деплой через Render

1. Загрузите **всю папку `mentoria-hub`** в один GitHub-репозиторий. В корне репозитория должны лежать `server.mjs`, `Dockerfile` и `render.yaml`.
2. В Render выберите **New → Blueprint** и подключите этот GitHub-репозиторий.
3. Render найдёт `render.yaml` и предложит создать сервис `mentoria-hub` и диск `mentoria-data`.
4. Заполните секретные переменные:
   - `ADMIN_EMAIL` — email главного администратора;
   - `ADMIN_PASSWORD` — уникальный пароль длиной от 12 символов;
   - `OPENAI_API_KEY` — необязательно, можно оставить пустым.
5. Подтвердите создание сервиса и дождитесь статуса **Live**.
6. Откройте выданный URL, войдите главным администратором и проверьте `/admin.html`.

Постоянный диск Render обычно требует платный экземпляр. Перед подтверждением Render покажет выбранный тариф. Если нужен полностью бесплатный вариант, JSON-хранилище следует заменить на внешнюю базу Supabase/Neon — это отдельная конфигурация.

## Как добавить ещё администраторов

1. Новый администратор сначала регистрируется на публичном сайте как обычный пользователь.
2. Главный администратор входит в `/admin.html`.
3. В списке пользователей нажимает **«Сделать админом»**.
4. После обновления страницы у назначенного пользователя появляется пункт **«Админ»** и доступ к общему управлению каталогом.

Все изменения, сообщения и аккаунты записываются в `/var/data/db.json` на постоянном диске и видны всем посетителям сайта.

## Обновления из GitHub

В конфигурации включён `autoDeploy`. После нового коммита в основную ветку Render пересоберёт сайт автоматически. Данные на подключённом диске при этом сохранятся.

## Перед публикацией

- Не используйте демонстрационный пароль из README.
- Проверьте официальные дедлайны программ ещё раз.
- Назначайте администраторами только доверенных людей.
- Включите резервные копии данных или периодически экспортируйте `db.json`.
