# Mentoria Hub 2.0

Полноценный многостраничный MVP по кейс-брифу Mentoria: реальная серверная регистрация и вход, персональный кабинет, каталог проверенных возможностей, курсы с прогрессом, общие чаты, AI-наставник и админ-панель.

## Быстрый запуск

Нужен Node.js 20 или новее. Зависимостей и `npm install` нет.

```bash
npm start
```

Откройте `http://localhost:5173`.

Для публичного сайта используйте готовые `render.yaml` и `Dockerfile`. Пошаговая инструкция находится в [DEPLOY.md](DEPLOY.md). Облачная конфигурация подключает постоянный диск, поэтому аккаунты, чаты и изменения администраторов сохраняются между обновлениями.

Демо-администратор при первом запуске:

- Email: `admin@mentoria.local`
- Пароль: `ChangeMeMentoria2026!`

Для реального размещения обязательно задайте собственные данные:

```powershell
$env:ADMIN_EMAIL="admin@example.com"
$env:ADMIN_PASSWORD="сложный-пароль-с-цифрой-2026"
npm start
```

## Что действительно работает

- Регистрация и вход через серверный API.
- Пароли хешируются с солью через `crypto.scrypt` и не возвращаются клиенту.
- Сессия хранится в `HttpOnly`, `SameSite=Lax` cookie.
- Данные пользователей, сохранения, прогресс, контент и сообщения записываются в `data/db.json`.
- Личный кабинет показывает сохранённые возможности, ближайший дедлайн, курсы и рекомендации.
- Каталог фильтруется по направлению, классу и статусу; прошедшие дедлайны определяются автоматически.
- Карточки ведут на реальные официальные сайты организаторов.
- Курсы можно добавить, а уроки — отмечать завершёнными.
- Три чат-комнаты получают новые сообщения через короткий polling; сообщения видны другим браузерам и сохраняются на сервере.
- Администратор может добавлять, редактировать и удалять возможности и видеть базовую статистику.
- Главный администратор может назначать другим зарегистрированным пользователям роль администратора.
- AI-наставник работает в умном demo-режиме без ключа. С `OPENAI_API_KEY` использует OpenAI Responses API на сервере.

## OpenAI (необязательно)

```powershell
$env:OPENAI_API_KEY="sk-..."
$env:OPENAI_MODEL="gpt-4.1-mini"
npm start
```

Ключ никогда не вводится в браузере и не попадает в клиентский JavaScript.

## Страницы

- `/` — главная
- `/auth.html` — регистрация и вход
- `/dashboard.html` — личный кабинет
- `/opportunities.html` — олимпиады и конкурсы
- `/opportunity.html?id=...` — подробности возможности
- `/courses.html` и `/course.html?id=...` — обучение и прогресс
- `/chat.html` — реальные общие чаты
- `/profile.html` — интересы и цели
- `/admin.html` — управление каталогом

## Безопасность и продакшен

Этот вариант подходит для хакатона, локального демо и небольшого пилота. Для настоящего запуска:

1. Замените JSON-хранилище на PostgreSQL / Supabase.
2. Вынесите сессии в Redis или таблицу сессий.
3. Добавьте подтверждение email и восстановление пароля через провайдера писем.
4. Настройте HTTPS, резервные копии, модерацию чатов и аудит действий администратора.
5. Не размещайте этот сервер на чисто статическом Netlify/Vercel-хостинге без отдельного backend и базы. Render, Railway, Fly.io или VPS подойдут для текущей архитектуры лучше.

```text

├── public/          # отдельные HTML-страницы, стили и клиентская логика

├── seed-data.mjs    # курсы, реальные возможности и стартовые сообщения
├── server.mjs       # HTTP-сервер, API, auth, persistence
└── package.json
```
